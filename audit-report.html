3. Alur Aplikasi

Login Flow

Login.jsx → api.login() → POST /api/auth/login → auth.controller.login()
→ bcrypt.compare() → generateToken() → response {token, user}
→ AuthContext.loginUser() → localStorage.setItem('token')
→ PrivateRoute cek user → render Layout + Dashboard

Data Flow

Tenants.jsx → useEffect → getTenants() → api.get('/tenants')
→ Vite Proxy → Express → authenticate middleware (cek JWT)
→ authorize middleware (cek role) → tenant.controller.getAll()
→ prisma.tenant.findMany() → SQLite → response JSON
→ setTenants(data) → React re-render

4. Daftar Fitur

Sudah Tersedia

  • Login & Register (tenant self-registration)
  • Dashboard (admin stats + tenant portal view)
  • CRUD Tenant (dengan kontak, catatan, dokumen)
  • CRUD Unit & Floor + Assign/Unassign tenant ke unit
  • CRUD Contract (dengan approval, termination, renewal)
  • CRUD Invoice (dengan bulk generate)
  • CRUD Payment (dengan verification)
  • Aging report, Notification system (cron reminder)
  • Tenant Portal (lihat tagihan, bayar)
  • Role-based access control, Search, filter, pagination
  • Responsive design (mobile sidebar)

Belum Selesai

  • Tidak ada unit test, integration test, atau API test
  • Tidak ada file upload (hanya URL string)
  • Tidak ada password reset, rate limiting, audit trail
  • Tidak ada backup mechanism, Docker, CI/CD
  • Zod, Multer, react-hot-toast di-declare tapi tidak digunakan

13. Bagian yang Sudah Baik

  1. Struktur modular — Backend feature-based module structure yang rapi.
  2. Auth middleware — authenticate dan authorize sudah benar.
  3. Role-based navigation — Frontend menyembunyikan menu berdasarkan role.
  4. PrivateRoute pattern — Protected routes dengan redirect ke login.
  5. Context API untuk auth — State management yang appropriate.
  6. Axios interceptor — Auto-attach token dan auto-redirect pada 401.
  7. Transaction usage — Operasi kompleks pakai prisma.$transaction.
  8. Pagination — Backend mendukung page dan limit parameters.
  9. UI consistency — Tailwind + custom components (Badge, Modal, StatCard).
  10. Seed data — Lengkap dengan data demo realistis.
  11. Cron job — Reminder system dengan node-cron.
  12. Aging report — Implementasi sudah benar.

15. Diagram Proses Bisnis

15.1 Tenant Onboarding

Admin buka Tenants → Klik Tambah → Isi form ↓ POST /api/tenants → System generate kode TNT-XXXX ↓ Tambah kontak (POST /api/tenants/:id/contacts) ↓ Assign ke unit (POST /api/units/:id/assign) ├── Unit tidak available → Error └── Unit available → TenantUnit + status occupied ↓ Buat kontrak (POST /api/contracts) ↓ Approve kontrak (PUT /api/contracts/:id/approve) ↓ Contract active + Tenant active → Selesai

15.2 Billing & Payment

Admin buka Billing ├── Manual → POST /api/billing └── Bulk → POST /api/billing/bulk-generate ↓ Generate invoice → Invoice + LineItem tersimpan ↓ Cron reminder → Notifikasi + Email ke tenant ↓ Tenant portal → Bayar → POST /api/tenant-portal/payments ↓ Payment status: pending_verification ↓ Admin verify → PUT /api/payments/:id/verify ├── Sudah lunas → Invoice status: paid └── Belum lunas → Invoice tetap pending

15.3 Contract Lifecycle

Buat Kontrak (draft) → Approve (active) ↓ Auto-generate invoice bulanan ↓ 3 bulan sebelum expired → Alert → Admin buat renewal ↓ Tenant accept → Update endDate + rent baru ↓ Expired → Status: expired ↓ Terminasi → Status: terminated → Exit date tercatat
